All errors thrown by Bora Pesa extend [`PesaError`](/docs/api/pesa/classes/PesaError). Use `instanceof` checks to handle specific error types.

## Error hierarchy

```
PesaError (base)
├── PesaValidationError    ← Bad input (amount ≤ 0, invalid phone, empty reference)
├── PesaNetworkError       ← Provider unreachable (DNS, timeout, connection refused)
├── PesaProviderError      ← Provider returned an error (HTTP 4xx/5xx, API error)
├── PesaWebhookError       ← Webhook signature verification failed
└── PesaUnsupportedError   ← Provider doesn't implement this optional operation
```

Every error has a `code` property for programmatic handling without `instanceof`:

| Error class             | `code`              |
| ----------------------- | ------------------- |
| `PesaValidationError`   | `VALIDATION_ERROR`  |
| `PesaNetworkError`      | `NETWORK_ERROR`     |
| `PesaProviderError`     | `PROVIDER_ERROR`    |
| `PesaWebhookError`      | `WEBHOOK_ERROR`     |
| `PesaUnsupportedError`  | `UNSUPPORTED`       |

## Handling errors

```ts
import { PesaValidationError, PesaNetworkError, PesaProviderError } from '@borapesa/pesa';

try {
  const order = await pesa.createOrder({
    amount:    15000,
    currency:  'TZS',
    reference: 'order_001',
    customer:  { name: 'Juma', phone: '255712345678' },
  });
} catch (err) {
  if (err instanceof PesaValidationError) {
    // Bad input — fix the payload. Don't retry.
    console.error('Invalid payload:', err.message);
    return { error: err.message };

  } else if (err instanceof PesaNetworkError) {
    // Provider unreachable — retry with backoff
    console.error('Network error:', err.message);
    await scheduleRetry('order_001');

  } else if (err instanceof PesaProviderError) {
    // Provider returned an error — check status code
    if (err.statusCode >= 500) {
      // Server error — retry
      await scheduleRetry('order_001');
    } else {
      // Client error — don't retry
      console.error('Provider rejected request:', err.message);
    }
  }
}
```

## When to retry

| Error | Retry? | Reason |
|---|---|---|
| `PesaNetworkError` | ✅ Yes | Transient — network will recover |
| `PesaProviderError` (5xx) | ✅ Yes | Provider is temporarily down |
| `PesaProviderError` (4xx) | ❌ No | Bad credentials, invalid request — retrying won't help |
| `PesaValidationError` | ❌ No | Fix the payload before resending |
| `PesaWebhookError` | ❌ No | Invalid signature — likely a forged webhook |
| `PesaUnsupportedError` | ❌ No | Provider doesn't support this operation — use feature detection |

## Feature detection

Some operations are optional — not all providers support refunds, previews, or cancellations. On the `PesaInstance`, unsupported operations are `undefined`. Check before calling:

```ts
// All optional operations support feature detection:
if (pesa.refund)              await pesa.refund('order_123', 5000);
if (pesa.cancelOrder)         await pesa.cancelOrder('order_123');
if (pesa.previewOrder)        await pesa.previewOrder({ ... });
if (pesa.previewDisburse)     await pesa.previewDisburse({ ... });
if (pesa.getBalance)          await pesa.getBalance();
if (pesa.getNameLookup)       await pesa.getNameLookup('255712345678');
if (pesa.validateCredentials) await pesa.validateCredentials();
if (pesa.listOrders)          await pesa.listOrders({ fromDate: new Date('2026-01-01') });
```

Calling an undefined operation throws `TypeError`, not `PesaUnsupportedError`. The provider-level default implementations throw `PesaUnsupportedError`, but `createPesa()` binds only supported methods — everything else is left `undefined`.
